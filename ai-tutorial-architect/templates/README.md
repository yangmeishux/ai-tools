## 🔄 使用流程

### 方式 1：基于任务分类使用（推荐）

```bash
# 1. 选择任务类型
[task:tutorial] 创建一个 React + TypeScript 完整教程

# 2. AI 自动选择对应模板
Chief 会参考"完整教程创作模板"的结构

# 3. 各 Agent 按模板协作
Educator 设计学习路径 → Writer 创作内容 → Practitioner 验证代码...

# 4. 输出符合模板规范的教程
```

### 方式 2：手动指定模板

```bash
# 1. 查看可用模板
> [@archivist] 列出所有可用的教程模板

# 2. 选择合适的模板
> [Chief] 使用"完整教程创作模板"创建 Python FastAPI 教程

# 3. 按模板结构执行
Chief 会按照模板定义的 Agent 协作流程执行
```

### 方式 3：自定义模板组合

```bash
# 组合多个模板
> [Chief] 创建教程：
  - 使用"概念指南模板"讲解原理部分
  - 使用"快速开始模板"编写入门章节
  - 使用"最佳实践模板"总结规范部分
  - 使用"速查表模板"提供命令参考
  - 最后使用"8维质量检查清单"验证
```

## 📝 自定义模板指南

### 创建自定义模板的步骤

1. **分析需求**：确定你的教程类型和目标受众
2. **选择基础**：基于现有模板进行修改
3. **定义 Agent 流程**：明确哪些 Agent 参与、执行顺序
4. **设置检查点**：在关键节点设置质量检查
5. **验证完善**：使用模板创作测试教程，不断优化

### 自定义模板示例

```markdown
---
template_name: 我的自定义模板
template_version: "1.0.0"
agents: [Chief, Writer, Editor]
task_type: custom
quality_level: silver
---

# 自定义模板结构

## 第一部分：[自定义内容]
[@researcher] 研究相关内容

## 第二部分：[自定义内容]
[@writer] 基于研究结果创作

## 质量检查
[@editor] 审查内容质量
```
