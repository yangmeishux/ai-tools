# 完整教程创作模板

> 适用于系统性、完整性要求高的技术教程
> 质量目标：🥇 金牌标准（8维质量评估全部通过）
> Agent 配置：Chief → Researcher → Educator → Writer → Practitioner → Learner-Advocate → Editor → [Humanizer] → Version-Archivist → Fact-Checker → Chief

---

## 🎭 Agent 协作流程

```
[Chief] 定义教程目标、受众群体、学习成果
    ↓
[@researcher] 调研技术主题、收集资料、发现常见问题
    ↓
[@educator] 设计学习路径、难度梯度、知识点拆解 ⚠️ 关键环节
    ↓
[@writer] 撰写教程内容、编写代码示例
    ↓
[@practitioner] 实际运行代码、验证可复现性 ⚠️ 关键环节
    ↓
[@learner-advocate] 从初学者角度审查、识别困惑点 ⚠️ 关键环节
    ↓
[@editor] 整合反馈、优化内容结构和表达
    ↓
[@humanizer] 去除 AI 痕迹、增加自然表达（可选但推荐）
    ↓
[@version-archivist] 标注技术栈版本、检查兼容性
    ↓
[@fact-checker] 验证技术细节、API 准确性和人性化后的内容正确性
    ↓
[Chief] 按照 8 维质量标准进行最终检查
    ↓
[@writer] 根据反馈进行最后修正
    ↓
[Chief] 批准并发布教程
```

## 📋 模板使用指南

### 适用场景
- ✅ 系统性技术课程（如"Python 全栈开发"）
- ✅ 框架/库的深度教程（如"React 18 完全指南"）
- ✅ 技术栈组合教学（如"MERN 栈实战"）
- ✅ 认证考试备考教程

### 不适用场景
- ❌ 简单的 API 使用说明
- ❌ 单次问题解决指南
