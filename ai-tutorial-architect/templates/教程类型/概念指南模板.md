# 概念指南模板

> 适用于深入讲解技术概念、原理、架构的内容
> 质量目标：🥈 银牌标准（注重准确性和深度）
> Agent 配置：Chief → Researcher → Educator → Writer → Fact-Checker → Editor → Chief

---

## 🎭 Agent 协作流程

```
[Chief] 明确概念范围和讲解深度
    ↓
[@researcher] 收集技术资料、研究原理、查找权威来源
    ↓
[@educator] 设计概念讲解结构、选择类比方式、确定难度梯度
    ↓
[@writer] 撰写概念讲解内容、编写示例代码
    ↓
[@fact-checker] 验证技术准确性、检查概念定义、核实数据来源
    ↓
[@editor] 优化表达结构、精炼语言、增强可读性
    ↓
[Chief] 最终审核并发布
```

---

## 📋 模板使用指南

### 适用场景
- ✅ 技术原理解析（如"深入理解 React Virtual DOM"）
- ✅ 架构设计讲解（如"微服务架构设计原则"）
- ✅ 算法/数据结构详解
- ✅ 设计模式深度剖析
- ✅ 新技术概念普及

### 不适用场景
- ❌ 操作步骤类教程
- ❌ API 使用文档
- ❌ 纯代码示例

### 核心特点
- 📐 **准确性**：概念定义准确，原理讲解无误
- 🎯 **深度**：不浮于表面，深入底层机制
- 🔄 **多角度**：原理 + 示例 + 类比 + 对比
- 💡 **易懂性**：复杂概念简单化
