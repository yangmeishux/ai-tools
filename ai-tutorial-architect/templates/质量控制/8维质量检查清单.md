# 8维质量检查清单

> AI 编程教程金牌标准质量评估工具
> 适用 Agent：Chief + 所有验证 Agent（Practitioner, Learner-Advocate, Version-Archivist, Educator, Fact-Checker）

---

## 📋 使用说明

### 何时使用
- 教程创作完成后进行质量评估
- 教程更新后的重新验证
- 多人协作时的质量把关
- 正式发布前的最终检查

### 检查流程

```
[Chief] 启动质量检查
    ↓
[@practitioner] 检查可运行性、环境完整性
    ↓
[@educator] 评估渐进性、实际价值
    ↓
[@learner-advocate] 审查术语解释、可复现性
    ↓
[@version-archivist] 确认版本明确
    ↓
[@practitioner + @learner-advocate] 共同检查错误处理
    ↓
[@fact-checker] 验证技术准确性
    ↓
[Chief] 汇总问题，生成改进清单
    ↓
[@writer] 根据清单修改内容
    ↓
[Chief] 重新检查，直到通过所有标准 ✅
```

## 维度 1：可运行性 (Runnability)

**检查者**：[@practitioner]

### 检查项

- [ ] **所有代码示例经过实际测试**
  - 每个代码块都被复制到实际环境中运行过
  - 没有语法错误
  - 没有运行时异常

- [ ] **能在指定的环境中成功运行**
  - 环境配置与教程描述一致
  - 依赖安装完整
  - 无缺失步骤

- [ ] **输出结果与文档描述一致**
  - 实际输出与教程中的"预期输出"匹配
  - 截图/示例结果真实有效

- [ ] **没有缺失的依赖或配置步骤**
  - 所有 import/require 的依赖都有安装说明
  - 配置文件都有创建/修改步骤

### 测试记录

| 代码片段 | 环境 | 测试结果 | 测试时间 | 测试人 |
|----------|------|----------|----------|--------|
| [示例1] | [环境] | ✅/❌ | [时间] | [姓名] |
| [示例2] | [环境] | ✅/❌ | [时间] | [姓名] |

### 问题记录

| 问题 | 位置 | 严重程度 | 修复方案 | 状态 |
|------|------|----------|----------|------|
| [问题1] | [章节] | 高/中/低 | [方案] | 待修复/已修复 |

## 维度 2：环境完整性 (Environment Completeness)

**检查者**：[@practitioner] + [@version-archivist]

### 检查项

- [ ] **明确列出所有依赖和版本号**
  - 主要工具版本明确（如 Python 3.11.2）
  - 依赖库版本明确（如 React 18.2.0）
  - 系统要求明确（如 Node.js >= 16）

- [ ] **提供完整的环境配置步骤**
  - 安装步骤清晰
  - 配置说明完整
  - 验证方法明确

- [ ] **标注操作系统兼容性**
  - Windows / macOS / Linux 支持情况
  - 特殊系统要求说明

- [ ] **说明前置条件（如已安装的工具）**
  - 读者需要预先掌握的知识
  - 需要预先安装的工具
  - 需要预先配置的环境

### 环境清单模板

```markdown
## 环境要求

### 硬件要求
- CPU：[最低] / [推荐]
- 内存：[最低] / [推荐]

### 软件要求
| 工具 | 版本 | 用途 | 检查命令 |
|------|------|------|----------|
| [工具1] | vX.Y.Z | [用途] | `[命令]` |
| [工具2] | vA.B.C | [用途] | `[命令]` |

### 前置知识
- [知识1]
- [知识2]
```

## 维度 3：渐进性 (Progressiveness)

**检查者**：[@educator]

### 检查项

- [ ] **难度梯度合理，从简单到复杂**
  - 开篇是简单的 "Hello World" 级别示例
  - 逐步引入复杂概念
  - 没有突兀的难度跳跃

- [ ] **每个步骤都有明确的学习目标**
  - 每章/每节开头说明学习成果
  - 学习者知道完成后的收获

- [ ] **避免一次性引入过多新概念**
  - 每个代码示例最多引入 1-2 个新概念
  - 新概念都有前置铺垫

- [ ] **提供练习题或实践任务**
  - 每章有配套的练习
  - 练习难度适中
  - 有参考答案或提示

### 难度评估

| 章节 | 难度 | 新概念数量 | 前置知识 | 评估 |
|------|------|------------|----------|------|
| [章1] | ⭐⭐ | 2 | [知识] | ✅/❌ |
| [章2] | ⭐⭐⭐ | 3 | [知识] | ✅/❌ |
| [章3] | ⭐⭐⭐⭐ | 2 | [知识] | ✅/❌ |

### 学习曲线

```
难度
  ▲
  │        ╭────
  │     ╭──╯
  │  ╭──╯
  │─╯
  └──────────────► 章节
```

## 维度 4：术语解释 (Term Explanation)

**检查者**：[@learner-advocate]

### 检查项

- [ ] **专业术语首次出现时有解释**
  - 每个专业术语首次出现都有定义
  - 解释通俗易懂

- [ ] **使用类比和生活化的例子**
  - 复杂概念有类比说明
  - 抽象概念有具体示例

- [ ] **避免"懂的人才知道"的隐含假设**
  - 没有跳过"显然"的步骤
  - 没有使用未解释的缩写
  - 没有引用未介绍的概念

- [ ] **提供术语表或索引**
  - 文档末尾有术语表
  - 或关键术语有超链接

### 术语检查表

| 术语 | 首次出现章节 | 是否有解释 | 解释质量 | 是否有类比 | 检查 |
|------|--------------|------------|----------|------------|------|

| [术语2] | [章2] | ✅/❌ | 好/中/差 | ✅/❌ | ✅/❌ |

## 维度 5：错误处理 (Error Handling)

**检查者**：[@practitioner] + [@learner-advocate]

### 检查项

- [ ] **标注常见错误和坑点**
  - 常见问题有专门的 FAQ 章节
  - 关键步骤有错误预警

- [ ] **提供错误排查步骤**
  - 错误有明确的排查流程
  - 排查步骤可操作

- [ ] **说明错误信息的含义**
  - 常见错误信息有解释
  - 错误原因有分析

- [ ] **给出解决方案**
  - 每个问题都有解决方案
  - 方案经过验证有效

### 常见问题模板

```markdown
### Q: [问题描述]

```
[错误信息或异常现象]

**原因**：
[根本原因分析]

**解决**：
```bash
[解决方案命令]
```

**预防**：
[如何避免这个问题]
```

## 维度 6：版本明确 (Version Clarity)

**检查者**：[@version-archivist]

### 检查项

- [ ] **标注所有技术栈的版本号**
  - 主工具版本明确
  - 依赖库版本明确
  - 系统环境版本明确

- [ ] **说明版本选择的原因**
  - 为什么选择这个版本
  - 新版本有什么特性
  - 旧版本为什么不推荐

- [ ] **提示版本兼容性问题**
  - 已知的不兼容情况
  - 版本升级注意事项

- [ ] **给出版本升级建议**
  - 如何升级到新版
  - 升级可能的风险
  - 迁移指南链接

## 维度 7：实际价值 (Practical Value)

**检查者**：[@educator] + [@practitioner]

### 检查项

- [ ] **解决真实世界的问题**
  - 教程基于真实场景
  - 不是纯理论或玩具示例
  - 学习后可以应用到工作中

- [ ] **提供可复用的代码模式**
  - 代码可以作为模板使用
  - 设计模式具有通用性
  - 解决方案可以迁移

- [ ] **说明应用场景和适用范围**
  - 明确说明适用场景
  - 明确说明不适用场景
  - 有真实的应用案例

- [ ] **避免过于简单或脱离实际**
  - 不是简单的 API 调用示例
  - 不是脱离实际的纯理论
  - 有完整的功能实现

### 价值评估

| 评估项 | 标准 | 评分(1-5) | 说明 |
|--------|------|-----------|------|
| 实用性 | 可应用到实际工作 | [1-5] | [说明] |
| 复用性 | 代码/方案可复用 | [1-5] | [说明] |
| 完整性 | 完整的解决方案 | [1-5] | [说明] |
| 时效性 | 解决当前问题 | [1-5] | [说明] |

## 维度 8：可复现性 (Reproducibility)

**检查者**：[@practitioner] + [@learner-advocate]

### 检查项

- [ ] **步骤清晰，无歧义**
  - 每个步骤都有明确的操作
  - 没有模糊的描述（如"适当调整"）
  - 命令可以直接复制执行

- [ ] **每个中间结果可验证**
  - 关键步骤有验证方法
  - 可以确认步骤执行成功
  - 失败时有排查方法

- [ ] **提供完整的示例代码**
  - 有完整的项目代码
  - 代码组织清晰
  - 有 README 说明

- [ ] **说明可能的变体和选项**
  - 可选配置有说明
  - 替代方案有介绍
  - 不同场景的适配方法
